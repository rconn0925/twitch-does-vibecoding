# Phase 1 — UI Design Contract

> Visual and interaction contract for the localhost **operator console** — the only UI surface in this phase. This is NOT the public stream overlay (that ships in Phase 2/3). Streamer-only, function over form, generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Auto-mode note:** This spec was compiled unattended as part of a `--chain` pipeline. All open design questions were resolved from `01-CONTEXT.md` decisions (D-01..D-18), the phase success criteria, and the explicit Claude's-discretion grant on console layout/aesthetics ("streamer-only, function over form"). No user questions were asked this session.

## Scope & Screens

The operator console (Express 5 + `ws`, bound to `127.0.0.1` only — never `0.0.0.0`) is a single-page, server-rendered utility surfaced across four views. This phase has **no component library and no design system tool** — it is hand-rolled semantic HTML/CSS/vanilla JS, consistent with the "plain `ws`, no framework" pattern CLAUDE.md already establishes for the overlay.

| Screen | Purpose | Locked decisions |
|--------|---------|-------------------|
| **Status bar (persistent, all screens)** | Current stream-mode pill (`IDLE`/`VOTING_ROUND`/`BUILD_IN_PROGRESS`/`FREE_REIGN_WINDOW`/`CHAOS_MODE`/`HALTED`) + always-visible Halt control | D-01, D-08 |
| **HALTED triage view** (replaces normal view when mode = HALTED) | Shows frozen in-flight task / queue / round state; operator picks Resume / Discard-Offending-Task-and-Resume / Reset-to-IDLE | D-02, D-04 |
| **Review queue** ("Needs Review") | Escalated (`held-for-review`) items; approve/reject per item, no active interruption | D-05, D-06, D-07, D-08 |
| **Active queue / task list** | Queued and in-progress tasks; per-task veto; force state transitions | D-01 |
| **Audit log** (Claude's discretion, scoped in) | Simple filterable table of `audit_log` + `review_queue` records | D-16, D-17, discretion note |

Every destructive/veto/halt control on every screen may optionally capture a one-tap reason tag (`ToS risk` / `boring` / `too big` / `gut feeling` / `other`) — tappable or skippable, never blocking (D-18).

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no shadcn/component-library. Tech stack (Express 5 + `ws`, server-rendered HTML) has no React/Next/Vite tier, so the shadcn init gate does not apply. |
| Preset | not applicable |
| Component library | none — hand-rolled semantic HTML + a single hand-written CSS file (`operator-console/public/console.css`). No JS framework; vanilla DOM + the existing `ws` reconnect pattern for live state push. |
| Icon library | none — text labels only. Status is communicated via the color-coded mode pill and status dots (`●`), not icon glyphs, to keep the dependency surface at zero. |
| Font | System font stack: `-apple-system, "Segoe UI", Roboto, sans-serif`. Native to the Windows 11 streaming host, zero network/dependency cost, appropriate weight for a utility console that must render instantly under incident conditions. |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/status-dot gaps, inline padding |
| sm | 8px | Compact element spacing (table cell padding, tag chips) |
| md | 16px | Default element spacing (form rows, card padding) |
| lg | 24px | Section padding (panel edges, screen margins) |
| xl | 32px | Layout gaps (between major console panels) |
| 2xl | 48px | Major section breaks (status bar to content) |
| 3xl | 64px | Page-level spacing (not expected to be used often on this dense console) |

**Exceptions:**
- All primary action buttons that carry incident-response weight (Halt, Veto, Approve, Reject, Resume, Discard & Resume, Reset to IDLE) get a **44px minimum click-target height**, even though the console is mouse-driven, not touch. Rationale: these are exactly the controls a streamer must hit correctly and fast under live-stream stress; oversized targets reduce mis-click risk during an actual incident.

## Typography

3-4 sizes, 2 weights, per contract:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 600 (semibold) | 1.5 |
| Body | 16px | 400 (regular) | 1.5 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

**Usage:**
- **Label** (14/600): table column headers, status-pill text, reason-tag chip labels, timestamps.
- **Body** (16/400): suggestion text, audit rationale text, form inputs, general console copy.
- **Heading** (20/600): section titles — "Needs Review", "Active Queue", "Audit Log", triage panel title.
- **Display** (28/600): the HALTED state banner headline only — this is the one piece of text that must be readable at a glance from across the room the instant the hotkey fires.

No third weight, no additional sizes. If a screen seems to need a 5th size, compress into Label or Body rather than adding a token.

## Color

60/30/10 split, dark utility theme (reduces glare next to an OBS-adjacent monitor; consistent with "function over form"):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F172A` (slate-950) | Page background, main content surface |
| Secondary (30%) | `#1E293B` (slate-800) | Panels, cards, table rows, status bar background |
| Accent (10%) | `#3B82F6` (blue-500) | Reserved for: primary interactive controls only — Approve button, Resume button, Discard-&-Resume confirm button, Reset-to-IDLE button, input focus rings, the currently-active mode pill's border. Never used decoratively. |
| Destructive | `#DC2626` (red-600) | Reserved for: Halt button (status bar + hotkey-triggered banner), Veto-task button, Reject button (review queue), the HALTED state banner background stripe. |

**Status-only extension (non-interactive, informational badges — does not count against the 10% interactive-accent budget):**

| Status | Value | Usage |
|--------|-------|-------|
| Approved | `#22C55E` (green-500) | Read-only `approved` decision pill in audit log / queue rows |
| Held-for-review | `#F59E0B` (amber-500) | Read-only `held-for-review` decision pill — D-08's honest three-state vocabulary must be visually distinguishable from approved/rejected at a glance |
| Rejected | reuses Destructive `#DC2626` | Read-only `rejected(category)` decision pill |

Accent reserved for (explicit list, never "all interactive elements"): Approve button, Resume button, Discard-&-Resume confirm button, Reset-to-IDLE button, focus rings, active-mode pill border.

Destructive reserved for (explicit list): Halt button, Veto-task button, Reject button, HALTED banner stripe.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (status bar, always visible) | **"Halt Everything"** — mirrors the panic hotkey; single click, no modal (D-03 places confirmation friction only on the hotkey's double-tap, explicitly "no other confirmation friction") |
| Secondary CTA — per-task | **"Veto Task"** on each queued/in-progress task row |
| Secondary CTA — triage (HALTED view) | **"Resume"** / **"Discard Task & Resume"** / **"Reset to Idle"** — three explicit buttons, no default/pre-selected option (D-04: nothing auto-resumes) |
| Secondary CTA — review queue | **"Approve"** / **"Reject"** per item |
| Empty state — review queue | Heading: **"Nothing waiting for review"** — Body: "Escalated suggestions land here between rounds. Nothing needs your attention right now." |
| Empty state — active queue (mode = IDLE) | Heading: **"Queue is empty"** — Body: "Waiting for the next round to start." |
| Empty state — audit log (filtered, no matches) | Heading: **"No matching records"** — Body: "Try widening your date range or clearing a filter." |
| Error state — classifier unavailable | Heading: **"Classifier unavailable"** — Body: "Retries exhausted — the suggestion was auto-rejected (fail-closed). Check `ANTHROPIC_API_KEY` and network status. No action needed here; this is logged automatically." |
| Error state — console disconnected | Heading: **"Console disconnected from orchestrator"** — Body: "Reconnecting… If this persists, restart the orchestrator process. The hotkey still works independently of this page." |
| Error state — invalid force-transition | Heading: **"Can't transition to {state} from {state}"** — Body: "Choose one of the valid recovery actions below." |
| Destructive confirmation — Halt Everything | No modal. Single click. Optional one-tap reason tag row appears immediately after (ToS risk / boring / too big / gut feeling / other / skip) — capturing it is optional and never blocks the halt, which has already taken effect (D-18). |
| Destructive confirmation — Veto Task | No modal. Single click, same optional reason-tag pattern as above. |
| Destructive confirmation — Discard Task & Resume | No modal (already gated behind explicitly entering the triage screen post-halt, per D-04). Single click, optional reason tag. |
| Destructive confirmation — Reject (review queue) | No modal. Single click — review queue is explicitly a low-friction, no-interruption workflow (D-05). |

**Design rationale:** CONTEXT.md's "Specifics" section is explicit that the user consistently chose low-interruption options (no attention pings, lean-reject, optional-not-mandatory reason tags) — the console is a between-rounds/incident-response surface, not a confirmation-modal-heavy admin panel. Every destructive action here is single-click by design; the *only* friction in the entire kill-switch system is the hotkey's own double-tap (D-03), and that friction must not be duplicated on the console.
