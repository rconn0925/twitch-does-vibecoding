## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Chat-derived vote identity → vote ledger | twitchUserId values originate from untrusted EventSub payloads (validated upstream in plan 02-04); the ledger's uniqueness key is the fairness-critical control |
| App → SQLite round/vote tables | Round state is the crash-recovery source of truth (D2-14) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | round_votes uniqueness key | mitigate | Key votes by Twitch numeric user id (chatterId), never display name — display names are mutable/spoofable via unicode tricks; PRIMARY KEY (round_id, twitch_user_id) + upsert makes one-vote-per-account structural (D2-15, RESEARCH.md Pitfall 2) |
| T-02-02 | Elevation of Privilege (of influence) | vote brigading via alt accounts | accept | One-vote-per-account + revote-override closes the basic case; account-age weighting is explicitly v2 (PITFALLS.md Pitfall 7); the round_closed audit row's tally JSON + the streamer veto on the queued winner are this phase's backstop |
| T-02-03 | Tampering | in-memory tally vs. durable ledger divergence on crash | mitigate | Write-through: SQLite upsert commits before the in-memory tally updates or VOTE_RECORDED emits; restore() rebuilds tally exclusively from round_votes (server-authoritative state, ARCHITECTURE.md Anti-Pattern 3) |
| T-02-04 | Denial of Service | halt vs. round-close timer race | mitigate | HALT_TRIGGERED handler cancels the timer synchronously and persists frozen_remaining_ms; closeRound() refuses the VOTING_ROUND→IDLE transition while HALTED (halt priority preserved, D-02/D2-16) |
</threat_model>
