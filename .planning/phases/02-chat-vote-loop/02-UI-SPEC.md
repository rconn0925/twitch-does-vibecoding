### Overlay screens/states (D2-18 — locked)

| State | What renders |
|-------|--------------|
| VOTING (round open) | Vote panel: round header + countdown timer, 2–3 numbered candidate rows with live tally bars; plus queue strip + state pill |

## Spacing Scale

Declared values (must be multiples of 4) — identical scale on both surfaces, inherited from Phase 1:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, pill padding |
| sm | 8px | Compact element spacing (tally count ↔ bar, strip chips) |
| md | 16px | Default element spacing (candidate row internals, form rows) |
| lg | 24px | Panel padding (vote panel edges, console panels) |
| xl | 32px | Gaps between overlay blocks (vote panel ↔ queue strip) |
| 2xl | 48px | **Overlay safe margin** — see exception below |
| 3xl | 64px | Page-level spacing (rare) |

**Exceptions:**
- **Overlay safe margin (auto-selected):** every overlay element keeps ≥48px (`2xl`) clearance from all four 1920×1080 canvas edges. 1080p action-safe is ~3.5% (~38px); 48px stays on-scale and survives OBS output cropping/scaling.
- **Console 44px minimum click-target** rule from Phase 1 carries forward to the new "Start Round" button (it is a live-show pacing control hit under stream pressure).
- The overlay has **no interactive elements**, so no touch/click-target exceptions apply there.

## Typography

Two surfaces, each with its own declared scale (3–4 sizes, 2 weights per surface):

### Operator console additions — inherit Phase 1 scale UNCHANGED

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 600 | 1.5 |
| Body | 16px | 400 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

New console elements map onto existing roles: "Start Round" button text = Body 16/600 (matches `.button-accent`), round countdown in the console status panel = Heading 20/600, disabled-reason text = Label 14/600 muted.

### Public overlay — NEW scale (auto-selected), sized for 1080p stream legibility

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 20px | 600 | 1.5 | State pill text, "NEXT UP" strip label, vote-command hint (`!vote 1/2/3`), tally counts |
| Body | 24px | 600 | 1.4 | Candidate titles (ballot rows), queue-strip item titles |
| Heading | 32px | 700 | 1.2 | Round header ("VOTE NOW"), winner announcement line |
| Display | 48px | 700 | 1.2 | Countdown timer digits only |

**Rationale (auto-selected):** nothing below 20px — smaller text smears under Twitch's video encode at 1080p and is illegible on the common 720p transcode. Minimum weight on the overlay is 600: regular-weight glyphs thin out over moving video. Exactly two weights (600, 700). If a 5th size seems needed, compress into Label or Body — never add a token.

## Color

### Operator console additions — inherit Phase 1 palette UNCHANGED

`console.css` custom properties (`--bg #0F172A`, `--panel #1E293B`, `--accent #3B82F6`, `--destructive #DC2626`, status green/amber) are reused as-is. **One delta:** the Phase 1 accent reserved-for list is extended by exactly one element — the **"Start Round" button** (primary interactive control, styled `.button-accent`). No other console color changes.

### Public overlay — NEW palette (auto-selected), same slate/blue family over transparent video

The page background is fully transparent (OBS browser source). The 60/30/10 split applies to the rendered chrome:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgba(15, 23, 42, 0.88)` (slate-950 @ 88%) | Vote panel background, queue strip background, state pill background — every text-bearing panel |
| Secondary (30%) | `rgba(30, 41, 59, 0.92)` (slate-800 @ 92%) | Tally bar tracks (unfilled), candidate number badges, queue-strip item chips |
| Accent (10%) | `#3B82F6` (blue-500) | Reserved for: **tally bar fills** and the **leading candidate's number badge**. Nothing else. |
| Text | `#F8FAFC` (slate-50) | All overlay text — one step brighter than the console's `#E2E8F0` to hold contrast over video bleed |
| Urgency (semantic) | `#F59E0B` (amber-500) | Reserved for: countdown digits during the final 10 seconds, and the "on hold" state-pill variant |
| Winner (semantic) | `#22C55E` (green-500) | Reserved for: the winning candidate's row highlight + winner checkmark text during the 8-second winner beat |

Accent reserved for (explicit list, never "all interactive elements"): tally bar fills; leading candidate's number badge. (The overlay has no interactive elements, so accent here is purely the "where your eye goes" 10%.)

Destructive `#DC2626` is **deliberately absent from the overlay**: D2-18 requires the HALTED display to stay honest but non-alarming on stream — the on-hold pill uses amber, never red. Red on the public surface would read as "the stream is broken."

**Legibility contract (hard rule):** no overlay text ever renders directly over video — every text node sits on a Dominant-color panel/pill background. No text-shadow hacks as a substitute for a backing panel.

### State pill variants (informational, outside the accent budget — Phase 1 precedent)

| Stream mode | Pill label (public wording) | Pill treatment |
|-------------|------------------------------|----------------|
| IDLE | `STANDBY` | Dominant bg, muted text `#94A3B8` |
| VOTING_ROUND | `VOTING OPEN` | Dominant bg, green-500 dot + slate-50 text |
| BUILD_IN_PROGRESS | `BUILDING` | Dominant bg, blue-500 dot + slate-50 text (a status dot, not an interactive accent — outside the 10% budget per Phase 1's status-extension rule) |
| HALTED | `ON HOLD` | Amber-500 dot + amber-500 text on Dominant bg — honest, non-alarming (D2-18) |

## Overlay Layout Contract (D2-18 specifics — Claude's discretion, auto-selected)

Canvas: 1920×1080, transparent background, 48px safe margin all edges.

| Block | Position | Size |
|-------|----------|------|
| **Vote panel** | Anchored lower-left (48px from left and bottom edges; queue strip clearance below handled by hiding the strip's left region or stacking panel above strip with 32px gap) | Fixed 560px wide; height fits content (header + 2–3 rows) |
| **Queue strip** ("next up") | Bottom edge, horizontal, right-aligned (48px from bottom/right) | One row, ≤3 queued items as chips; hidden entirely when queue is empty |
| **State pill** | Top-right corner (48px from top/right) | Single pill, Label 20/600 |

**Vote panel anatomy (top to bottom):**
1. Header row: `VOTE NOW` (Heading 32/700) left, countdown `0:47` (Display 48/700, tabular-nums) right; beneath the header a vote-command hint `type !vote 1, 2 or 3` (Label 20/600, muted).
2. Candidate rows (2 or 3, per D2-04): number badge (`1`/`2`/`3`, Secondary bg — Accent bg on the current leader) + candidate title (Body 24/600, single line, ellipsized) + tally bar (full row width beneath title, 12px tall track in Secondary, fill in Accent) + vote count right-aligned (Label 20/600, tabular-nums).
3. Winner beat (8s after close): winning row gets green-500 left border + `WINNER` label; losing rows dim to 50% opacity; countdown area shows `Round over`.

**Chat-derived text safety (hard rules — chat text is broadcast on stream):**
- All chat-derived strings (candidate titles, everywhere on both surfaces) render via **`textContent` only — never `innerHTML`** (extends the Phase 1 machine-checked invariant to `overlay/public/`; the single-funnel test's zero-innerHTML scan must cover the new directory).
- **Truncation limits:** candidate titles truncate to **80 characters** (JS-side, append `…`) on vote-panel rows and **60 characters** on queue-strip chips — plus CSS `white-space: nowrap; overflow: hidden; text-overflow: ellipsis` as the visual backstop. Suggestions can be up to 2000 chars (parser bound); untruncated titles would break the layout on stream.
- No user-controlled URLs, images, or markup are ever interpreted — titles are opaque plain text.

**Motion & update cadence (from 02-RESEARCH.md Pitfall 3):**
- Server pushes tally diffs on a **300ms debounce** while a round is open (auto-selected within the researched 250–500ms band) — never per-vote.
- Tally bar width animates with CSS `transition: width 300ms ease-out` so debounced jumps read as smooth motion, not flicker.
- Countdown renders **client-side** from a server-sent `endsAtMs` timestamp (1s tick, local interpolation, resynced on every state push) — the server never streams per-second timer frames.
- On ws disconnect, the overlay **freezes its last-known render and retries silently** with backoff. **No error text ever renders on the broadcast surface.** On reconnect it receives full state and re-renders (OBS scene-switch reloads get the same full-state-on-connect treatment — D2-18).

## Copywriting Contract

### Operator console (new elements)

| Element | Copy |
|---------|------|
| Primary CTA (this phase) | **"Start Round"** — accent button in the console's round panel; single click, no modal (round start is not destructive) |
| Start Round disabled (pool < 2, per D2-04) | Button disabled + Label text: **"Need at least 2 approved candidates to start a round (have {n})."** |
| Start Round disabled (round already open) | Button disabled + Label text: **"Round in progress — {m:ss} left."** |
| Start Round disabled (mode ≠ IDLE) | Button disabled + Label text: **"Rounds can only start from Standby (current mode: {mode})."** |
| Round status panel (round open) | Heading: **"Round open — {m:ss} left"** — body lists numbered candidates with live counts (the console MAY show live tallies; only chat may not — D2-07) |
| Round status in HALTED triage (D2-16) | Line inside the existing frozen-state triage panel: **"Round frozen at {m:ss} remaining — {n} votes recorded. Resume continues the round; Discard cancels it and returns candidates to the pool."** |
| Empty state — round panel (no round, pool has candidates) | Heading: **"No round running"** — Body: "{n} candidates ready. Start a round when the show's ready for one." |
| Empty state — candidate pool | Heading: **"Candidate pool is empty"** — Body: "Waiting for !suggest ideas from chat. Approved ideas appear here." |
| Error state — Twitch disconnected | Heading: **"Twitch connection lost"** — Body: "Reconnecting automatically (no action needed). Votes sent during the gap can't be recovered; the tally resumes from the last saved state." |
| Error state — Twitch auth expired | Heading: **"Twitch login expired"** — Body: "Re-authorize at /auth/start to reconnect chat. Rounds can't run until this is fixed." |
| Destructive actions this phase | **None new.** Halt Everything and Veto Task carry over from Phase 1 unchanged (single click, optional reason tag, no modal). Start Round is not destructive and gets no confirmation friction. |

### Public overlay (on-stream copy)

| Element | Copy |
|---------|------|
| Round header | **"VOTE NOW"** |
| Vote hint | **"type !vote 1, 2 or 3"** (renders "1 or 2" in a 2-candidate round) |
| Winner beat | **"WINNER"** label on the winning row; header area reads **"Round over"** |
| Queue strip label | **"NEXT UP"** |
| State pill labels | `STANDBY` / `VOTING OPEN` / `BUILDING` / `ON HOLD` (see State pill variants above) |
| Empty states | Silent by design: no round → vote panel absent; empty queue → strip absent; the state pill is the only permanent element. Never render "no data" placeholder text on stream. |
| Error states | None rendered — freeze-and-silently-retry contract (see Motion & update cadence). |

### Bot chat narration (CHAT-05 / COMP-03 / D2-06 / D2-07 — chat copy is UI copy)

Tone: game-show beat, terse, no lectures. All messages go through the single rate-limited sender queue (D2-08). **Candidate titles inside chat messages truncate to 60 characters** — Twitch caps messages at 500 characters and a 3-candidate round-open message must always fit.

| Event | Template |
|-------|----------|
| Round open | `Voting is OPEN — !vote 1, 2 or 3: [1] {title} [2] {title} [3] {title} — {duration}s on the clock.` |
| Round close + winner | `Round over — "{title}" wins with {n} votes. Queued for the build.` |
| Round close + tiebreak (D2-03) | `Dead heat! Coin flip says… "{title}". Queued for the build.` |
| Round close, zero votes | `No votes this round — candidates return to the pool. We'll run it back.` |
| Rejection feedback (category-level, COMP-03) | `@{user} that one can't run on stream: {category label}. Different idea?` |
| Rejection — feasibility trim (D-13 variant) | `@{user} too big for a live build — try a smaller version of that idea.` |
| Held for review (D-08 vocabulary) | `@{user} that idea is held for streamer review — it may join a later round.` |
| Duplicate suggestion (D2-12) | `@{user} that one's already in the pool — vote for it when it comes up!` |
| Cooldown notice (D2-11, coalesced) | `@{user} easy there — one suggestion per {n}s.` |
| Coalesced multi-rejection (D2-07 burst rule) | `@{user1} {category label}. @{user2} {category label}. @{user3} held for review.` (one message, semicolon-free @-runs, still ≤500 chars) |
| Error (public-facing, rare) | `Suggestion check is backed up — hold your ideas for a minute, votes still count.` |

Category labels come from the existing viewer-safe `CATEGORY_META` labels in `src/compliance/categories.ts` — chat never sees internal category codes or classifier rationale.
