<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host ↔ WSL2 distro | Untrusted, chat-driven build execution runs inside the distro; secrets and the orchestrator live on the host |
| WSL2 distro ↔ Windows loopback services | NAT mode must keep this one-directional (host→sandbox only) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | WSL2 drvfs automount exposing `/mnt/c` | mitigate | automount disabled in `/etc/wsl.conf` + escape-attempt test (Task 2a) proves `cat /mnt/c/...` fails |
| T-03-02 | Spoofing | Sandbox reaching host loopback-trusted console/overlay via mirrored networking | mitigate | keep default NAT mode; Task 2b proves sandbox cannot reach 127.0.0.1:4900/4901 |
| T-03-03 | Denial of Service (of the kill switch) | Hung Linux process surviving a veto | mitigate | `wsl.exe --terminate` teardown proven against a deliberately hung tree (Task 2c) |
| T-03-04 | Information Disclosure | Host `claude login` credential copied into the sandbox | mitigate | separate in-distro `claude login`; host `~/.claude/` never crosses the boundary (Task 1.5) |
| T-03-05 | Tampering (billing) | Metered per-token billing silently enabled in-sandbox | accept-with-verification | A1 confirmed against billing surface (Task 2d); fallback = sandbox-scoped key only |
</threat_model>
