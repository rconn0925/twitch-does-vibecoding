## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chat text → agent prompt | Untrusted chat-derived text crosses into the agent's context; must stay data, never instruction |
| agent SDK surface → src/ | The `query()`/`spawnClaudeCodeProcess` tool-use authority must be reachable from ONE module tree only (src/orchestrator/), not any file that imports the SDK |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Elevation of Privilege | Chat-driven prompt injection reaching the agent's tool-use authority | mitigate | fixed orchestrator-authored system prompt + delimited-data user turn (prompt-boundary.ts); adversarial-fixture invariant suite proves it |
| T-03-09 | Tampering | A future orchestrator file bypassing the boundary by interpolating text into a prompt | mitigate | source-scan invariant (single-funnel discipline) fails the build on any such interpolation |
| T-03-12 | Elevation of Privilege | A file OUTSIDE src/orchestrator/ importing `@anthropic-ai/claude-agent-sdk` (`query`) and gaining a second, un-boundaried path to agent tool-use authority (SAND-04 boundary; previously convention-only) | mitigate | agent-SDK-confinement source scan (mirrors single-funnel check (c)) asserts the SDK/`query` import lives only under src/orchestrator/; synthetic-offender self-test + non-empty-scan guard prove the scan is live |
