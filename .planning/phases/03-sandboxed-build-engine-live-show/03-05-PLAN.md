Build the WSL2 sandbox adapter (the phase's only process-sandboxing code) and the veto-teardown primitive. |----------|-------------|
| host process.env → sandbox spawn env | Host secrets must never be forwarded into the spawned WSL2 engine's environment |
| streamer veto → hung sandbox process tree | The kill must reach the Linux tree, not just the wsl.exe wrapper |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Information Disclosure | Host secrets (TWITCH_*, .env) forwarded into the sandbox env | mitigate | explicit env allowlist (PATH only) in sandbox-process.ts; secrets-isolation source-scan invariant enforces it |
| T-03-13 | Denial of Service (of the kill switch) | wsl.exe orphaning — hung Linux tree survives a veto | mitigate | sandboxTeardown = `wsl.exe --terminate <distro>` in the abort fan-out (BUILD-04), proven vs a hung tree in Wave 0 |
| T-03-14 | Tampering | Silently-unsandboxed build if the sandbox layer is unavailable | mitigate | `sandbox.failIfUnavailable: true` — fail loud, never run unsandboxed |
| T-03-04 | Information Disclosure | In-distro Anthropic auth conflated with host secrets | accept (documented) | separate in-distro `claude login`; if A1 false, a distinct SANDBOX_ANTHROPIC_API_KEY only — never the host key |
