<objective>
Ship the new app-under-construction preview surface (PRES-03): a separate browser/OBS source that frames the sandboxed dev server and auto-refreshes, so viewers watch the app come alive. Per D3-12 it is strictly isolated — it holds NO orchestrator connection, subscribes to no orchestrator state, and renders zero chat-derived text; its only dynamic input is whether the dev server on `127.0.0.1:<PREVIEW_DEV_SERVER_PORT>` (reachable from the host via WSL2 NAT localhostForwarding) responds. An unreachable server reads as a calm "STANDING BY" with a friendly placeholder — never a browser error page, never red.

Purpose: The live app is the payoff of the whole loop; it must be viewable without exposing anything of the host or orchestrator (SAND-02).
Output: `src/preview/server.ts` (+test), `src/preview/preview-manager.ts` (+test), and `src/preview/public/` (html/css/js).
</objective>
