<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| trigger event -> FSM | Amount/donor from an untrusted upstream event (validated at 04-02) drives duration + window grant |
| FSM -> single funnel | Donor instruction crosses into the queue ONLY via the injected funnel (gate re-screen happens there) |
| in-memory FSM <-> durable row | Crash boundary — ends_at_ms is the source of truth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Tampering | A large donation creating an unbounded/monopolizing window | mitigate | amountToDurationSeconds hard cap (maxSeconds) + one-active-window + per-donor cooldown (D-04/D-05), all unit-asserted |
| T-04-08 | Tampering | Crash re-arming a fresh full-duration timer (donor gets more time than paid) | mitigate | restore() derives remaining from absolute ends_at_ms; test asserts re-arm==remaining, never full duration (RESEARCH Pitfall 2) |
| T-04-09 | Elevation of Privilege | Paid window enqueuing directly, bypassing classify() | mitigate | control-window.ts never imports gate/queue; only the injected funnel (04-06, allowlisted) can enqueue — structurally enforced |
| T-04-10 | Tampering (race) | Concurrent trigger opening a second window between check and write | mitigate | synchronous guard-then-transition with no await between the mode/cooldown check and the state write (single-threaded event loop); flagged as an explicit review point per RESEARCH Security Domain |
| T-04-11 | Denial of Service | A window that fails to close/expire (stuck FREE_REIGN_WINDOW) | mitigate | fail-closed: revoke() always available (reuses HALT machinery); restore() closes any orphaned expired row on boot |
</threat_model>
