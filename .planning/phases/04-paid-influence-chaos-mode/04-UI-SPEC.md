# Phase 4 — UI Design Contract

> Visual and interaction contract for **paid influence & chaos mode**: the public OBS overlay gains a free-reign window banner, two new (calm) state-pill words, and build-provenance chips; the operator console gains a control-window panel with Revoke, a chaos-mode toggle, a donations connection pill, and new audit event filters. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Auto-mode note:** Compiled unattended inside an autonomous chain. Every open design question was resolved from `04-CONTEXT.md` (D-01..D-10), `04-RESEARCH.md` (StreamElements `tip` shape, EventSub redemption shape, ControlWindow FSM, T-04 "narrow public projection" threat note), the shipped Phase 2/3 overlay (`02-UI-SPEC.md`, `03-UI-SPEC.md`, `src/overlay/public/`), and the shipped Phase 1/3 console (`01-UI-SPEC.md`, `src/operator-console/public/`). Auto-selected values are marked "(auto-selected)". No user questions were asked this session.

## Scope & Surfaces

Two surfaces, both **extensions** of shipped surfaces — they inherit their existing contracts unchanged; this spec declares only the deltas. This phase EXTENDS the established design system; it invents nothing new except one semantic color (violet, justified below).

| Surface | Posture | Governing rules |
|---------|---------|-----------------|
| **Public OBS overlay** (`src/overlay/public/`) — EXTENDS Phases 2/3 | Read-only broadcast surface at 1920×1080. Adds: a **free-reign window banner** (donor + countdown, top-left), two new pill words (`FREE REIGN` / `CHAOS`), and a **provenance chip** on the build panel. Coarse, honest, never red, never an error string. The donor's donation **message text never renders on stream** — donor display name (truncated) + countdown only (T-04 narrow-public-projection, CONTEXT "coarse public surface"). | Phase 2/3 overlay contract inherited unchanged; deltas below |
| **Operator console additions** (`src/operator-console/public/`) — EXTENDS Phases 1/3 | Streamer-only, function over form, full honest detail. ### Overlay state model (extends D2-18 / Phase 3's state table)

Phase 2 locked `STANDBY / VOTING OPEN / BUILDING / ON HOLD` as "the ONLY state words that ever reach the stream" — that lock held because `FREE_REIGN_WINDOW`/`CHAOS_MODE` were unreachable (mapped to `STANDBY` in `PILL_BY_MODE`). Phase 4 makes both modes reachable, so this spec performs the **one deliberate vocabulary extension**: the pill set becomes exactly six words. No other state word may ever be added without a UI-SPEC amendment.

| StreamMode | Pill (updated `PILL_BY_MODE`) | Overlay presentation |
|------------|------------------------------|----------------------|
| `IDLE` | `STANDBY` | unchanged (Phase 2) |
| `VOTING_ROUND` | `VOTING OPEN` | unchanged (Phase 2) |
| `BUILD_IN_PROGRESS` | `BUILDING` | Build panel (Phase 3) **+ provenance chip** when the task came from a window or a chaos pick; **free-reign banner stays mounted** if the active window is still ticking |
| `FREE_REIGN_WINDOW` | **`FREE REIGN`** (new) | Free-reign banner (top-left): `FREE REIGN` label + donor display name + m:ss countdown. Vote/build panels absent (windows open from IDLE; the lower-left slot is empty until the donor's instruction starts building) |
| `CHAOS_MODE` | **`CHAOS`** (new) | Pill only — no chaos panel. The chaos pick itself is narrated in chat and shows on-stream as the build panel mounting with a `CHAOS PICK` chip (auto-selected: no dedicated overlay "pick beat"; the pick→build transition is near-instant and a beat would collide with the build panel mounting) |
| `HALTED` | `ON HOLD` | unchanged (Phase 2) — a revoked window that came from a halt shows `ON HOLD`, amber, never red |

**Banner independence rule:** the free-reign banner is driven by `OverlayState.controlWindow` (non-null while a window is active), NOT by the pill mode — so it persists across the `FREE_REIGN_WINDOW → BUILD_IN_PROGRESS` transition while the donor's build runs, and collapses on expiry/revoke regardless of mode. The pill and the banner never disagree about time: both derive from the same pushed `endsAtMs`.

### Public overlay additions — inherit Phase 2/3 palette + ONE new semantic color

Page background stays fully transparent. Existing tokens unchanged. One addition:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgba(15, 23, 42, 0.88)` | Free-reign banner background (same as every text-bearing panel) |
| Secondary (30%) | `rgba(30, 41, 59, 0.92)` | Build-panel provenance chip background |
| Accent (10%) | `#3B82F6` | UNCHANGED — active pipeline-stage badge / tally fills / leader badge only. **No Phase 4 element uses the accent.** |
| Text | `#F8FAFC` | Banner donor name + countdown digits |
| Success (semantic) | `#22C55E` | unchanged |
| Urgency (semantic) | `#F59E0B` | unchanged + the banner countdown's final-10-seconds digits (consistent with the vote countdown) |
| **Paid-control (semantic, NEW)** | `#8B5CF6` (violet-500) | **Reserved exclusively for:** the `FREE REIGN` pill dot, the banner's `FREE REIGN` label + dot, and the build-panel `FREE REIGN` provenance chip text. Nothing else, ever. |

**Why a new color (auto-selected):** a paid window is a headline show mechanic that must read as its own thing at a glance — green is voting, blue is building, amber is hold/urgency. Violet-500 stays inside the same Tailwind-500 family as the rest of the palette, is calm (not alarming) over video, and gives paid control a single ownable hue. It is a **semantic** color like green/amber, not an interactive accent.

**Chaos gets NO color (auto-selected, deliberate):** the `CHAOS` pill uses a **slate-50 (`#F8FAFC`) white dot** with default pill text, and the `CHAOS PICK` chip uses default text on Secondary. Zero new hue, and — mirroring D-08's code-path separation — paid control and chaos **never share a visual identity**: violet means "someone paid for control," white/neutral means "randomness." No element may ever style a chaos state in violet or a paid state in the chaos treatment.

**Destructive `#DC2626` remains deliberately absent from the overlay** (carried forward, hard rule). A revoked or expired window collapses calmly — no red, no error string, no "revoked" wording on stream. Red on the broadcast surface reads as "the stream is broken."

**Legibility contract (hard rule, carried forward):** no overlay text renders directly over video — the banner is a Dominant-color panel like everything else.

## Overlay Free-Reign Banner — Layout Contract (PAID-01/02/04, auto-selected)

A new persistent element mounted while `OverlayState.controlWindow !== null`. It is the ONLY overlay surface for an active window — coarse by design.

| Property | Value |
|----------|-------|
| Position | Anchored top-left, 48px from top and left edges (the unused corner — never collides with pill/panels/strip) |
| Size | max-width 640px; height fits content |
| Background | Dominant `rgba(15,23,42,0.88)`, 12px radius, 8px 16px padding |
| Layout | Single row, 16px gaps: violet dot (12px) → `FREE REIGN` (Label 20/600, violet `#8B5CF6`) → donor display name (Body 24/600, slate-50) → countdown (Heading 32/700, tabular-nums, slate-50) |

**Content rules (hard):**
- **Donor display name is attacker-controlled** (StreamElements `displayName` / EventSub `user_name`): renders via **`textContent` only — never `innerHTML`** (the zero-innerHTML invariant scan must cover this code path), **JS-truncated to 24 characters** (append `…`) + CSS `nowrap/ellipsis` backstop.
- **The donation/redemption message text NEVER renders on the overlay** — not truncated, not sanitized: absent (coarse public surface; the instruction reaches the stream only later as the build-panel task title after it has cleared the gate).
- **No amount, no currency, no duration-mapping math on stream** (auto-selected): donor name + remaining time only. The amount→duration mapping is the console's and audit ledger's job (PAID-04).
- Both trigger types (donation and channel-points) use the identical banner and the same `FREE REIGN` wording — one FSM, one presentation (D-03). No separate "points" vocabulary on stream.

**Countdown behavior:** computed client-side from the pushed `endsAtMs` on the existing 1s tick (mirrors the vote countdown — the server never streams timer frames; resynced on every push). Digits flip to Urgency amber for the final 10 seconds. On OBS scene-switch reload, full-state-on-connect reconstructs the banner exactly.

**Expiry / revoke (never red, never narrated on-screen):** the banner collapses on the next push (`controlWindow: null`). No "revoked"/"expired" text ever renders on the broadcast surface — the chat narration and audit ledger carry the never-silent duty (doctrine: transitions → chat, state → overlay). On ws disconnect the banner freezes its last render and retries silently with backoff (carried forward).

## Operator Console — Control-Window Panel & Chaos Toggle (PAID-04, CHAOS-01)

### Control-window panel

A new `<section id="window-panel" class="panel" hidden>` directly above the round panel (build panel ordering unchanged). Hidden entirely when no window is active (matches the build panel's hidden-not-empty posture). While a window is active:

1. **Heading** (Heading 20/600, tabular-nums): **"Free-reign window — {m:ss} left"** — client-side 1s tick from `endsAtMs`, resynced per push (reuses the round-panel countdown machinery).
2. **Donor line** (Body 16/400): "{displayName} — donation" or "{displayName} — channel points". Full name shown (console is private); `overflow-wrap: anywhere`.
3. **Mapping line** (Body 16/400, muted): the honest PAID-04 ledger view — **"$5.00 → 1:00 window (capped at 5:00)"** for donations / **"{cost} points → 0:30 window"** for redemptions. The console always shows the amount→duration math the overlay deliberately hides.
4. **Revoke Window** button — destructive red, 44px min target, **single click, no confirmation modal** (Phase 1 doctrine: the only friction anywhere is the hotkey's own double-tap). After the click, the existing D-18 reason row appears (optional one-tap tag, never blocking, `kind: "revoke-window"`).

### Chaos-mode toggle

Lives in the **round panel** (it swaps the same selection concern Start Round owns). One accent button, 44px min target, single click, no modal:

- Chaos off → button label **"Enable Chaos Mode"**. Disabled (with a `.round-reason` muted line) unless mode is `IDLE`: "Chaos can only start from Standby (current mode: {mode})."
- Chaos on → button label **"Disable Chaos Mode"** (same accent styling — it's the same mode-change control), plus a panel body line: **"Chaos mode on — the next task is a random pick from the approved pool. No vote runs."** Start Round is disabled with reason: "Voting is off while chaos mode is on."

Precedence surfacing (D-05): while a window is active, the chaos toggle is disabled with reason "A free-reign window is active — it takes precedence."

### Status bar additions

- **Donations pill** next to the existing Twitch pill: `Donations: connected / reconnecting / not configured` (colors per the Color section). When `reconnecting`, an error-state box appears (copy below). `not configured` shows the pill only — expected pre-setup state, no error box.
- **Twitch pill fourth state**: `Twitch: missing channel-points scope` (amber) + error box (copy below).
- The mode-pill strip already contains `FREE_REIGN_WINDOW` and `CHAOS_MODE` — the existing active-border treatment covers them with zero changes.

### Audit log additions (COMP-05 / PAID-04 / never-silent)

The event-type filter dropdown gains: `window_opened`, `window_expired`, `window_revoked`, `window_denied`, `chaos_toggled`, `chaos_pick`. Window rows populate the existing columns (Source = `donation`/`channel_points`/`chaos`, User = donor identifier, Suggestion/Rationale = the amount→duration mapping text and denial reason). Table structure, columns, and styling unchanged.

### Bot chat narration (never-silent doctrine — chat copy is UI copy)

Follows the Phase 2/3 rate-budget doctrine (transitions → chat, high-frequency state → overlay) and game-show-beat tone. All messages go through the single rate-limited sender queue. Task titles truncate to 60 characters; donor names are Twitch/SE handles (≤25 chars, no truncation needed). Window/chaos events are low-frequency — well inside the rate budget.

| Event | Template |
|-------|----------|
| Window opened (donation) | `@{donor} tipped {amount} and takes the wheel — free reign for {m:ss}! Type !build <your instruction> to use it.` |
| Window opened (channel points) | `@{user} redeemed {reward} — direct control for {m:ss}! Type !build <your instruction> to use it.` |
| Window denied — one already active (D-05, dropped/queued with feedback) | `Thanks @{donor} — a control window is already running, so this one can't open. One window at a time.` |
| Window denied — donor cooldown | `Thanks @{donor} — you're on cooldown from your last window. Try again in a bit.` |
| Instruction rejected by the gate (PAID-03 — narrated, never silent, window time not consumed) | `Can't build that one, @{donor} — it didn't pass the safety check ({category label}). Your window's still open — try another idea.` |
| Instruction held for review | `That one needs a human look first, @{donor} — the streamer's checking it. Your window's still open.` |
| Instruction accepted | `Locked in — building @{donor}'s pick: "{title}".` |
| 30 seconds remaining (only for windows ≥ 60s) | `30 seconds left on @{donor}'s window.` |
| Window expired | `Time's up — @{donor}'s window is closed. Back to the regular show.` |
| Window revoked | `Streamer's call — the control window is closed early. Back to the regular show.` |
| Chaos on | `CHAOS MODE ON — the next build is a random pick from the approved pool. No votes.` |
| Chaos off | `Chaos mode off — voting is back.` |
| Chaos pick | `Chaos pick: "{title}" — no vote needed, building it now.` |

**Copy separation invariant (mirrors D-08, hard rule):** paid-window copy **never** mentions chance, luck, odds, rolls, or randomness; chaos copy **never** mentions money, tips, donations, points, or paying. No message may ever imply that paying buys a chance at anything — payment buys guaranteed, time-boxed control; chaos is free and random. This applies to every string in this contract and any future addition. (`!build` as the window command token is auto-selected; the planner may rename it, but the narration templates must be updated in lockstep.)

Category labels come from the viewer-safe `CATEGORY_META` labels (`src/compliance/categories.ts`) — chat never sees internal codes or classifier rationale (carried forward).
