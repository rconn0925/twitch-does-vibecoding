<objective>
Close the provenance-wiring gap and add the durable persistence that HIST-01's changelog reads from. Today provenance is never threaded to build-completion (`BuildStatusView.source` is dead code in production; the three `main.ts` drivers never pass provenance into `startBuild()`), and no ledger row can losslessly reconstruct a changelog entry (D-01 VERDICT). This plan adds a dedicated append-only `build_history` table + a `recordBuildHistory`/`listBuildHistory` pair, and threads provenance from the three build-trigger sites through `startBuild()` → `finalize()` → the new record.

This is the foundation/seam wave. It writes no served surface — it makes the data spine exist and correct, so 05-02 can render it.

Purpose: every completed build leaves an honest, gate-approved, provenance-tagged, append-only record that survives restarts (SQLite durability) — the single source of truth the changelog queries.
Output: `build_history` table + record/list functions + shared types + the finalize() hook + provenance threaded at three call sites.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chat-derived suggestion text → durable build_history row | Only gate-APPROVED QueuedTask.text may be persisted; raw pre-gate text must never enter this table |
| build lifecycle → append-only ledger | A completed build persists a record; an aborted/vetoed build must not |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | build_history containing pre-gate / banned suggestion text (D-03) | mitigate | recordBuildHistory is called ONLY from finalize() with task.text off an already-branded QueuedTask (single-funnel gate-approved); no call site passes raw SuggestionCandidate.text; append-only (no UPDATE/DELETE) so a row cannot later be mutated to carry disallowed text |
| T-05-02 | Repudiation | An aborted/vetoed build silently recorded as a "built" outcome (false BUILT-IT record) | mitigate | recordBuildHistory is NOT called in finalizeAborted() (CR-01); test asserts zero rows on abort; stage→result mapping is 1:1 and honest (done→built, failed→failed, refused→refused) |
| T-05-03 | Tampering | Provenance mis-attribution (a paid/chaos build recorded as vote) | mitigate | explicit provenance threaded from each of the three drivers rather than mode-inferred; driveWindowBuild reads the live ControlWindowSnapshot.trigger; unit-tested per provenance |
| T-05-SC | Tampering (supply chain) | package installs | accept | no new package installs in this plan — reuses better-sqlite3/zod already vetted in Phases 1-4 |
</threat_model>
