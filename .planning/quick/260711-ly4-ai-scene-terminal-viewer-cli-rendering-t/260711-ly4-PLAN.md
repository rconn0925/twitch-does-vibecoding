<!-- Verified against the codebase — the executor should use these directly. -->

**The wire (src/overlay/server.ts):** There is NO dedicated /builder WebSocket.
The overlay server runs ONE `WebSocketServer` on the same HTTP server
(`http://127.0.0.1:${OVERLAY_PORT ?? 4901}`, see src/main.ts:1786). Every
connected client receives the FULL `OverlayState` JSON: once on connect, then
on every change push (including pushes triggered by votes/pool/rounds where
`builderFeed` is unchanged — the CLI must treat those as no-ops). The builder
feed is the `builderFeed` field:

```typescript
// OverlayState (the only field this CLI reads):
builderFeed: { kind: string; text: string }[];

// BuilderFeedLine kinds (CLOSED set, src/overlay/builder-feed.ts):
kind: "title" | "stage" | "stage-warn" | "activity" | "snippet";
```

**WS handshake:** `verifyClient` accepts non-browser clients: Origin absent is
OK, but the Host header MUST be a loopback host (`isLoopbackHostHeader`).
Connect to `ws://127.0.0.1:${port}` with the `ws` package (already a
dependency) — default headers pass this check.

**Feed semantics to mirror (src/overlay/public/builder.js + builder-feed.ts):**
- Server ring buffer holds max 50 lines; `buildStarted()` CLEARS the buffer
  and appends `{ kind: "title", text: "NOW BUILDING: <title…>" }`. So a feed
  whose first line is a NEW title ⇒ clear the terminal view, fresh header.
- Snippet text may contain up to 3 `\n`-separated lines, ≤200 chars + ellipsis
  (server-authoritative caps). Client backstop caps in builder.js: 120 chars
  per non-snippet line, 200 for snippets.
- Empty `builderFeed` ⇒ idle ("standing by…" in builder.js).
- Unknown kind ⇒ render NOTHING (fail closed).
- stage-warn is amber, never red (D2-18). Disconnect ⇒ freeze last render,
  silent backoff retry (builder.js: `Math.min(500 * 2 ** attempts, 8000)`).
