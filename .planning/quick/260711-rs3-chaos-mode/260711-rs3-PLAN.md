<objective>
Chaos Mode (Ross directive 2026-07-11): a timed, chat-activated show mode. When CHAOS_ACTIVATION_VOTES unique chatters type `!chaos`, chaos mode activates for CHAOS_MODE_DURATION_SECONDS, then auto-reverts to democratic mode. While active, the auto-cycle suggest window runs unchanged, but at window close the vote round is skipped — one random pooled candidate is enqueued through the SAME winner path a voted win uses (the q5n kind router in drainVoteQueue). Surfaces: overlay badge (CHAOS MODE, slate-50 dot, m:ss countdown), server-composed narration beats, audit rows.

Purpose: chat gets a pressure valve to skip democracy for a bounded window — without touching the single funnel, the paid↔chance decoupling, or halt semantics.
Output: `!chaos` parser+dispatch, `src/chaos/mode.ts` controller, vote-skip pick hook in the auto-cycle, overlay wire+badge, narration beats, audit rows, e2e + invariant proofs.
